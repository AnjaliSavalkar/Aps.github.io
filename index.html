Problem 1: Two Sum

Problem Statement:

Given an array of integers nums and an integer target, return indices of the two numbers such that they add up to target.

You may assume that each input would have exactly one solution, and you may not use the same element twice.

Objectives:

  • Implement an efficient algorithm to find two numbers that sum to the target value
  • Optimize the solution for time complexity
  • Handle edge cases gracefully
  • Demonstrate understanding of hash maps and their application in algorithmic problem solving

My Approach:

I implemented a hash map solution with O(n) time complexity, where n is the length of the input array. This approach allows us to find the complementary value in constant time, significantly improving performance over a brute force approach.

Problem 2: Longest Substring Without Repeating Characters

Problem Statement:

Given a string s, find the length of the longest substring without repeating characters.

Objectives:

  • Implement a sliding window algorithm to track unique characters
  • Optimize for both time and space complexity
  • Handle edge cases such as empty strings and strings with all repeating characters
  • Demonstrate understanding of string manipulation techniques

My Approach:

I used a sliding window technique with a hash set to track characters in the current window. This approach achieves O(n) time complexity where n is the length of the string, while maintaining O(min(m,n)) space complexity where m is the size of the character set.

Problem 3: Merge K Sorted Lists

Problem Statement:

You are given an array of k linked-lists lists, each linked-list is sorted in ascending order. Merge all the linked-lists into one sorted linked-list and return it.

Objectives:

  • Implement an algorithm to efficiently merge multiple sorted linked lists
  • Apply appropriate data structures (e.g., priority queue/heap) to optimize the merging process
  • Analyze and optimize the time and space complexity of the solution
  • Demonstrate understanding of linked list operations and heap data structure

My Approach:

I implemented a solution using a min-heap (priority queue) to track the current minimum node from each list. This approach achieves O(N log k) time complexity, where N is the total number of nodes across all lists and k is the number of linked lists.
